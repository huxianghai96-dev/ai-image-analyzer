# 约束与决策说明

## 技术选型

在开发机仅有 Python + OpenCV、无 Node/Flutter SDK 的条件下，选用 **Python + Flet** 作为移动端方案：Flet 基于 Flutter 渲染，可用 Python 编写 UI，并支持打包 Android APK，满足「移动端工具」要求，同时保证作业可复现。

## 内存与性能

12MP 原图直接分析易 OOM 且耗时超标。决策：**分析流水线统一将最长边缩放至 1280px**（预览可至 1920px），在文档中说明大图耗时策略。实测 4000×3000 JPEG 分析约 103ms，远低于 1s 目标。

## 算法取舍

采用传统 CV（Laplacian、直方图、Immerkaer 噪点、RMS 对比度），放弃深度学习语义质量评估——作业明确不要求模型，且可避免 APK 体积膨胀与 GPU 依赖。

## 格式支持

实现 JPEG/PNG/WebP/HEIC 四种。HEIC 依赖 `pillow-heif`，在部分 Windows 环境安装较慢；若缺失则给出明确错误提示，不静默失败。

## 放弃项

- 完整相册浏览、批量分析、历史记录
- RAW/ TIFF / AVIF 等专业格式
- 美观 UI 动画与主题定制
- 云端同步与分享

## 已知局限

Laplacian 清晰度对噪点/锐化伪影敏感（见反例）；已在综合层做噪点交叉校正，但无法完全消除原始 Laplacian 虚高；曝光仅基于全局直方图，不处理局部 HDR；噪点估计在强纹理场景可能偏高。综合分通过多维度加权缓解单一指标失准。
