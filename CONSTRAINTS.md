# 约束与决策说明

## 技术选型

在开发机仅有 Python + OpenCV、无 Node/Flutter SDK 的条件下，选用 **Python + Flet** 作为移动端方案：Flet 基于 Flutter 渲染，可用 Python 编写 UI，并支持打包 Android APK，满足「移动端工具」要求，同时保证作业可复现。

## 移动端适配

选用 Flet 内置的 **FilePicker** 组件实现图片选择，在 Android 上映射为系统原生文件选择器。同时保留路径输入框作为桌面调试的备用入口。分析流程放到后台线程执行，避免 UI 阻塞。

## 内存与性能

12MP 原图直接分析易 OOM 且耗时超标。决策：**分析流水线统一将最长边缩放至 1280px**（预览可至 1920px），在文档中说明大图耗时策略。实测 4000×3000 JPEG 分析约 122ms，远低于 1s 目标。

## 算法取舍

采用传统 CV（Laplacian、直方图、Immerkaer 噪点、RMS 对比度），放弃深度学习语义质量评估——作业明确不要求模型，且可避免 APK 体积膨胀与 GPU 依赖。深度学习（pyiqa + cnniqa）作为可选加分项保留。

## 打包取舍

APK 使用 `build_apk.py` 构建，在 Flet 生成后清理 `.git`、测试样本、个人样本和缓存文件，防止交付包体积异常或泄露非交付素材。

## 格式支持

Android APK 主链路实现 JPEG/PNG/WebP 三种，满足「至少 3 种格式」要求。HEIC 依赖 `pillow-heif`，该库没有适合当前 Flet Android 打包器的 wheel，因此保留为桌面扩展能力；若运行环境缺失 codec，会给出明确错误提示，不静默失败。

## 放弃项

- 完整相册浏览、批量分析、历史记录
- RAW / TIFF / AVIF 等专业格式
- 精美 UI 动画与主题定制（保持信息层次清晰即可）
- 云端同步与分享
- 本地持久化分析记录（内存中仅保存当前会话）

## 已知局限

Laplacian 清晰度对噪点/锐化伪影敏感（见 3 类反例）；已在综合层做噪点交叉校正，但无法完全消除 Laplacian 虚高。曝光仅基于全局直方图，不处理局部 HDR。噪点估计在强纹理场景可能偏高。对比度权重仅 15%，在雾天等低对比度场景中综合分拉低力度不足。综合分通过多维度加权缓解单一指标失准。
